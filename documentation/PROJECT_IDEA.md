Слой 1 — Документ идеи проекта
PROJECT: Логистический центр (Delivery Operations System)
1. ❗ Проблема (с цифрами)
Текущая ситуация (реальный кейс)

Компания:

10 курьеров
~100 заказов/день
доставка по зонам + км
Как сейчас работает:
маршруты строятся через Яндекс Маршрутизация
выплаты считаются частично вручную
мотивация фиксирована и не адаптивна
🔴 Конкретные боли:
1. Переплата за логистику
стоимость: ~650 000 руб / год за маршрутизацию
10–30% переплата курьерам из-за:
неэффективных маршрутов
плохого распределения заказов

👉 Потери:
≈ 100 000 – 300 000 руб / месяц

2. Время логиста
планирование: 2–4 часа в день
переработка маршрутов: ещё 1–2 часа

👉 Потери:
≈ 80–120 часов / месяц

3. Текучка курьеров

Причины:

“много езжу — мало получаю”
непрозрачная система выплат
перегруженные маршруты

👉 Потери:

найм + обучение = 5–20К / курьер
нестабильность SLA
4. Отсутствие прозрачности

Руководитель НЕ видит:

прибыль по доставке
эффективность курьеров
эффективность логистов
реальную стоимость заказа
2. 💡 Решение (конкретный процесс)
Система работает так:
Шаг 1 — Приём данных
CRM → API → система
данные:
заказы
адреса
временные окна
параметры заказа
Шаг 2 — Анализ

Система:

считает расстояния
анализирует зоны
анализирует загрузку курьеров
оценивает SLA
Шаг 3 — Генерация маршрутов
авто-распределение заказов
учёт:
геозон
км
времени
загрузки
Шаг 4 — Назначение курьеров
система предлагает распределение
логист подтверждает / редактирует
Шаг 5 — Выполнение

Курьер:

получает маршрут в приложении
выполняет шаги
отправляет статусы
Шаг 6 — Расчёт выплат

Система:

считает выплаты автоматически
учитывает:
зоны
км
бонусы
штрафы
Шаг 7 — Аналитика

Руководитель видит:

стоимость доставки
эффективность
отклонения
KPI
Шаг 8 — AI рекомендации

Система предлагает:

изменить зоны
изменить оплату
изменить загрузку
3. ⏳ Почему сейчас
1. Рынок
рост e-commerce
рост last-mile доставки
2. Проблема не решена
Яндекс Маршрутизация решает только маршруты

3. Технологии
