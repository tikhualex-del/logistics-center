## Important: Prisma 7 Breaking Change

Prisma 7 (installed: 7.7.0) removed the `url` field from `datasource db` in `schema.prisma`. Connection URL is now managed exclusively in `prisma.config.ts`.

For `PrismaClient` instantiation in the application, pass the URL via constructor:
```typescript
// If needed explicitly (usually auto-reads from prisma.config.ts in CLI, but runtime reads .env via @nestjs/config)
new PrismaClient({
  datasources: { db: { url: process.env.DATABASE_URL } }
})
```

The current `PrismaService` relies on NestJS `ConfigModule` loading `.env` before the service starts, which is standard behavior with `ConfigModule.forRoot({ isGlobal: true })`.
